=== Betrieb der Anwendung

==== Start und Stopp der Anwendung

Starten der Anwendung:

- Melden Sie sich am Server mit administrativen Rechten an.
- Starten Sie den {application}-Dienst.
- Prüfen Sie, ob der Webserver und die Datenbankverbindung aktiv sind.
- Testen Sie den Zugriff auf die Weboberfläche über den Browser.

Beenden der Anwendung:

- Informieren Sie die Benutzer über die geplante Abschaltung.
- Fahren Sie den {application}-Dienst geordnet herunter.
- Beenden Sie ggf. geplante Hintergrundjobs und sichern Sie offene Datenbanktransaktionen.

==== Benutzerverwaltung / Rollen 

Die Benutzerverwaltung erfolgt zentral in der Administrationsoberfläche der Applikation {application}. 

- Anlegen, Bearbeiten und Löschen von Benutzern
- Zuweisen von Rollen wie „admin“ oder benutzerdefinierte Rollen 
- Festlegen individueller Rechte für bestimmte Funktionsbereiche (z. B. Automatisierung)
- Regelmäßige Überprüfung aktiver Benutzer und Rechte (mindestens halbjährlich)

==== Anbindung an andere Systeme (Schnittstellen, APIs)

Die Anwendung ist über folgende Schnittstellen und APIs mit anderen Systemen verbunden:

- RAG-API zur Kommunikation mit dem Vector-Store
- LLM-API zur Kommunikation mit dem vom Kunden konfigurierten LLM Provider
- Schnittstelle mit vom Kunden konfigurierte Authentifizierung
- (optional) MCP Server API zur Nutzung kundenspezifizierten Suchfunktionen

Die Schnittstellen werden regelmäßig auf ihre Funktionsfähigkeit geprüft und dokumentierte Änderungen am API-Standard werden frühzeitig implementiert.

==== Backup und Recovery der Applikation

Backup und Recovery der Applikation erfolgt durch den Kunden

==== Überwachung/Monitoring

Überwachung/Monitoring erfolgt durch den Kunden
