=== IT-Sicherheit und Datenschutz

==== Zugriffsschutz und Berechtigungsmanagement

Der Zugriff auf {application} ist durch ein mehrstufiges Berechtigungssystem sowie technische Schutzmaßnahmen abgesichert:

- Anmeldung erfolgt ausschließlich über das vom Kunden gewählte und konfigurierte Anmeldeverfahren (über die lokale Benutzer-/Passwortverwaltung
(LDAP des Kunden, DB des Kunden), über Azure AD/Entra (OIDC/SAML), über API-Token für Integrationen)
- Passwortrichtlinien entsprechen den kundeneigenen IT-Sicherheitsvorgaben (Mindestlänge, Komplexität, regelmäßiger Wechsel).
- Der Zugriff auf sensible Daten und Administrationsfunktionen ist ausschließlich ausgewählten Benutzerrollen vorbehalten.
- Verschlüsselung des Datenverkehrs zwischen Client und Server durch TLS 1.2 oder höher.

==== Security und Compliance

===== TLS/HTTPS

Obligatorisch für Produktion. Zertifikate:

- Let's Encrypt (via Caddy)
- DNS-01 (API-Token Provider)
- Eigene Zertifikate (interne CA)

===== Firewall

**Ausgehende Verbindungen:**

- **Immer:** Docker Hub / CGS Registry
- **LLM-Provider:** Je nach Anbieter
- **Lokale Modelle:** Interne Verbindung

===== Datenschutz

**Anwendungsdaten:** 
- Ausschließlich auf Kundenserver

**LLM-Provider (Cloud):**

- Azure OpenAI / AWS Bedrock: Anfragen werden übertragen, aber nicht dauerhaft gespeichert
- **Empfehlung:** EU-Region wählen (DSGVO)

**Lokale Modelle:** 

- Höchste DSGVO-Konformität

==== Protokollierung und Logging

Es werden Protokolle (Logs) zur Sicherstellung von Nachvollziehbarkeit geführt:

- Die Logfiles werden automatisch zentral archiviert in der SQLite DB.
- Logdaten werden durch den Kunden gesichert und Datenschutzkonform behandelt.
